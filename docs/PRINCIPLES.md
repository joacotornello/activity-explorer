# Product principles
## Accessibility is a first-class feature
The Activity Feed must be fully operable with keyboard and screen reader. Focus must be visible and never "lost" under sticky UI.

## Clarity beats cleverness
Default experience: show "what happened recently" (newest first), with obvious ways to filter by author and color.

## Explore first, edit never
This app is an explorer for board activity. It does not attempt to replicate the board editor.

## No meaning by color alone
Color is a helpful cue but must always be paired with text labels and/or iconography.

## Engineering principles
**Single source of truth for server state**
All note data is requested via typed API functions and cached via query keys. Infinite pagination is managed by the query layer. Backend returns limited amount of data via pagination.

## UI state is shareable and debuggable
Filters live in the URL query string. Selection can be local.

## UI copy has one source of truth
User-facing text should come from a centralized English text module instead of being hardcoded across components. This keeps labels, loading states, empty states, error messages, and accessibility text consistent without introducing localization infrastructure in v1.

## Tokens-only design system boundary
The tokens layer is generated by Style Dictionary and exported to the app as CSS variables for simplicity. Components are built in the app layer.

## Optimize only where it matters
Virtualize the Activity Feed (largest DOM risk) and paginate requests to backend.
